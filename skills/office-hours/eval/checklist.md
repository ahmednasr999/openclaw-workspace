# Office Hours Eval Checklist

## Pass/Fail Criteria

| # | Check | Pass | Fail |
|---|-------|------|------|
| 1 | Triggered only on multi-step or ambiguous requests | ✅ | Triggered on simple lookup |
| 2 | Asked about the problem, not the feature | ✅ | Asked "what do you want me to build?" |
| 3 | Reframed when answers revealed different need | ✅ | Built exactly what was asked despite signals |
| 4 | Brief written with all 5 sections | ✅ | Missing sections or no brief |
| 5 | Conversation felt natural, not interrogative | ✅ | Questions felt like a checklist |
| 6 | Skipped when request was clear single-step | ✅ | Over-applied to trivial request |

## Scoring

- 6/6: Perfect
- 5/6: Good
- 4/6: Acceptable
- ≤3/6: Needs improvement

## Example Failures

**Over-trigger:** User says "check my calendar for tomorrow" → Agent asks "what problem are you trying to solve?"
→ FAIL (question 6)

**Under-reframe:** User says "build a script to send emails" but describes a CRM need → Agent builds email script
→ FAIL (question 3)

**Interrogation mode:** Agent responds with numbered list: "1. What problem... 2. What does success... 3. What's the risk..."
→ FAIL (question 5)

# Premium Frontend Patterns

## Executive dashboard
Use when the user needs decision speed.
- top summary strip with 3-5 key metrics
- dense but calm tables
- separators over excessive cards
- tabular numbers
- restrained color, often one risk/accent color
- sticky filters only if they save real work

Avoid:
- decorative hero sections
- animated metric counters
- unnecessary glassmorphism

## Internal tool / operations UI
Use when workflows matter.
- clear left-to-right task flow
- persistent status and next action
- compact forms with strong labels
- inline validation
- empty states that teach the workflow

Avoid:
- hiding primary actions in kebab menus
- vague button labels

## SaaS/product landing page
Use when persuasion matters.
- strong hero with specific audience and outcome
- proof section with realistic metrics/logos/testimonials
- asymmetric feature section instead of three generic cards
- visible CTA rhythm every 1-2 sections
- crisp product screenshots or abstracted UI mockups

Avoid:
- “seamless next-gen platform” copy
- purple-blue AI gradients by default
- meaningless icon cards

## Premium editorial page
Use when brand perception matters.
- generous whitespace
- strong typography pairing
- one memorable visual motif
- restrained palette
- fewer sections, higher quality

Avoid:
- too many effects
- stock-photo cliché
- huge decorative labels like “SECTION 01”

## Redesign move set
If the current UI is generic, apply these in order:
1. Tighten typography scale and line-height.
2. Fix container width and section spacing.
3. Reduce accent colors to one.
4. Replace equal-card monotony with an asymmetric or hierarchy-led layout.
5. Add missing states and focus styles.
6. Add subtle texture/depth only after the basics work.

# Pattern: Thesis Summary

## Best for
- executive summary
- key recommendation
- opening the body of the deck with the main conclusion

## Use when
- one core thesis should lead the page
- 2 to 4 supporting points justify or sharpen that thesis

## Don’t use when
- the slide is just listing all findings equally
- there is no clear dominant conclusion yet

## Required content inputs
- main thesis or recommendation
- 2 to 4 supporting statements

## Optional content inputs
- one proof point
- one key metric
- one short implication line

## Layout logic
- dominant thesis block first
- support items clearly secondary
- proof point may act as a subordinate accent, not a competing hero

## Hierarchy rules
- thesis must dominate
- support points should be concise
- any metric should reinforce the thesis, not hijack it unless this should actually be a metric slide

## Focal-point rule
The focal point is the thesis statement. If support items compete with it, reduce them.

## Visual treatment guidance
- strong headline scale
- asymmetry preferred
- use one accent moment only where it clarifies the recommendation

## Common anti-patterns
- summary as bullet dump
- all support points equal weight
- too much body text
- no clear recommendation or conclusion

## Example structures
- left thesis, right support stack
- top thesis, lower supporting evidence band
- thesis plus one subordinate proof tile

## QA checklist
- can the viewer state the main recommendation after 3 seconds?
- is the thesis obviously dominant?
- could one support item be removed to make the slide sharper?

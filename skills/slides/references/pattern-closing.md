# Pattern: Closing

## Best for
- final recommendation
- call to action
- end-state takeaway
- decisive closing page

## Use when
- the deck needs to end with one clear takeaway or next move
- the audience should leave with a memorable final frame

## Don’t use when
- the slide is just administrative housekeeping
- the deck has no actual decision, recommendation, or closure

## Required content inputs
- final takeaway, recommendation, or next step

## Optional content inputs
- one supporting reason
- one deadline or action owner
- one closing visual field

## Layout logic
- closing thought should dominate the page
- supporting content should be minimal
- the page should feel poster-like, not like another content slide

## Hierarchy rules
- final statement first
- supporting line second
- meta or logistics last

## Focal-point rule
The focal point is the closing statement. Everything else exists only to sharpen that final impression.

## Visual treatment guidance
- bold ending is encouraged
- use whitespace generously
- if using accent, use it to reinforce the final action or recommendation

## Common anti-patterns
- ending with a weak summary list
- closing page that feels operational instead of decisive
- too many next steps packed into the final slide

## Example structures
- centered final statement with small action line
- left-weighted recommendation with bold accent field
- closing poster with one supporting proof line

## QA checklist
- does the deck end decisively?
- is the closing statement memorable?
- could any supporting detail be removed?

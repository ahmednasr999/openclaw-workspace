# Pattern: Comparison

## Best for
- option tradeoff
- before / after
- current state vs target state
- decision framing

## Use when
- the audience needs to compare 2 to 4 structured alternatives
- contrast is the main communication job

## Don’t use when
- one side is obviously the recommendation and the other side is only context, unless the design makes that dominance clear
- the content would work better as a metric or thesis-summary slide

## Required content inputs
- two or more compared items
- comparison basis or lens

## Optional content inputs
- recommendation
- one winning criterion highlight
- risk / tradeoff note

## Layout logic
- symmetry only when the comparison genuinely demands it
- otherwise let the recommended or preferred option dominate subtly
- keep comparison axes clear and limited

## Hierarchy rules
- comparison frame first
- compared items second
- recommendation or verdict, if present, should resolve the slide clearly

## Focal-point rule
The focal point is either the comparison tension itself or the recommended side, not decorative cards.

## Visual treatment guidance
- use contrast, spacing, and scale to clarify differences
- avoid over-boxing
- if the slide has a recommendation, let the composition admit it

## Common anti-patterns
- fake symmetry when one option is clearly preferred
- too many dimensions in one slide
- equal emphasis on all options with no conclusion
- side-by-side grids with no visual hierarchy

## Example structures
- dominant recommendation column + secondary alternative
- before / after split with asymmetrical emphasis
- two-option comparison with explicit decision strip

## QA checklist
- is the comparison basis obvious?
- does the composition reflect the real decision weight?
- is there unnecessary symmetry?

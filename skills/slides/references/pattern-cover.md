# Pattern: Cover

## Best for
- opening the presentation
- setting tone and ambition
- establishing the deck’s design language

## Use when
- this is slide 1
- the deck needs a strong first impression
- there is one headline idea that can carry the page

## Don’t use when
- the slide is mostly agenda or navigation
- there are too many operational details that belong on slide 2

## Required content inputs
- main title

## Optional content inputs
- subtitle
- presenter / organization
- date or event context
- supporting visual field or image

## Layout logic
- one strong dominant title block
- subtitle and meta are clearly subordinate
- use asymmetry by default unless the concept strongly supports center alignment
- allow generous negative space

## Hierarchy rules
- title should dominate the slide
- subtitle should be meaningfully smaller
- meta should feel quiet and precise

## Focal-point rule
The focal point is usually the title block or the title paired with one visual field. Do not create multiple competing hero elements.

## Visual treatment guidance
- oversized type is encouraged
- one visual anchor only
- restrained accent usage
- do not crowd the cover with logos, badges, or multiple devices unless the brief truly requires it

## Common anti-patterns
- logo soup
- too many small text elements
- weak title scale
- centered “safe” layout with no tension
- cover that looks like a normal content slide

## Example structures
- left-weighted title with right visual field
- centered title over image or texture field
- quiet typographic poster cover

## QA checklist
- does the title dominate immediately?
- is there enough whitespace?
- does the cover feel more like a poster than a template?

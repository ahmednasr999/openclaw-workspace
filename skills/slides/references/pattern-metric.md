# Pattern: Metric

## Best for
- KPI story slides
- one-number proof slides
- performance or outcome emphasis

## Use when
- one metric is the story
- the slide exists to make a number matter

## Don’t use when
- several metrics are equally important
- the message requires a full comparison grid or richer chart story

## Required content inputs
- metric value
- metric label
- time or context frame

## Optional content inputs
- delta vs baseline
- source note
- one-line implication

## Layout logic
- the metric should dominate the page
- explanatory copy should remain subordinate
- charting is optional and should support, not overwhelm

## Hierarchy rules
- number first
- label and context second
- explanation third

## Focal-point rule
The focal point is the number or metric block. If secondary elements become equally loud, remove them.

## Visual treatment guidance
- large data typography is encouraged
- use accent to support the metric, not decorate surrounding boxes
- avoid unnecessary gauges and dashboard chrome

## Common anti-patterns
- too many KPIs on one slide
- oversized chart when the number is the story
- metric hidden among cards or bullets

## Example structures
- giant KPI left, implication right
- centered hero metric with small support band
- one metric plus baseline comparison strip

## QA checklist
- does the number dominate immediately?
- is the message about the metric clear?
- would removing the chart make the slide stronger?

# Visual Explainer

Use this skill for turning complex ideas into simple visual explanations, diagrams, or image concepts.

## Process

1. Clarify the single message the visual must communicate.
2. Choose the simplest format: comparison, flow, ladder, matrix, timeline, or before/after.
3. Keep copy short and executive-readable.
4. Match Ahmed's premium, practical, GCC/executive positioning when used for public content.
5. Verify the output visually or by direct file inspection before declaring it ready.

## Safety

- Do not publish or send externally without explicit approval.
- Avoid fabricated data, logos, endorsements, or claims.
- If the visual workflow becomes recurring, skillify it into a deterministic template or generator.

assistant: 

Before 2026.3.2, memory search embeddings could use: OpenAI, Gemini, Voyage, Mistral, or local GGUF models (which is what QMD tries and fails on our VPS).

Now there's a new option: **Ollama**. It's a local LLM server that runs embedding models efficiently on CPU, without the heavy llama.cpp compilation mess QMD hits.

**Why this could matter for us:**

| Current problem | Ollama solution |
|---|---|
| QMD local embeddings fail (no GPU, slow compile) | Ollama runs pre-built, no compilation |
| We set QMD to BM25-only as a workaround | Ollama could give us real vector search back |
| Remote embeddings cost money (tiny, but non-zero) | Ollama is fully local and free |

**But honestly?** For our setup, I'd park this. We just set QMD to BM25 search mode and it works. Adding Ollama means installing another service, managing another process, using more VPS memory. Our 2-core VPS is already running OpenClaw, PM2, Mission Control, and crons.

If you ever move to a beefier VPS or want semantic memory search without paying for remote embeddings, Ollama becomes the right answer. For now, BM25 keyword search handles our memory recall fine.

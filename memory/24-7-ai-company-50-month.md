## The Problem

### The 9-Agent Disaster
- Built 9 AI agents ("dream team")
- Burned $100 in ONE NIGHT on API calls
- Got banned from Antigravity for rate limit violations
- Total damage: $200+ in 3 days

### Why It Failed
- 9 agents all pulsing 24/7 = API hits every few seconds
- Google token limits obliterate fast
- Rate limits hit = bans = fallbacks = more money

---

## The Solution

### The $50 Power Move
- Fired all "Pro" tools (Claude Pro, Gemini Pro)
- Replaced with: M4 Mac Mini + MiniMax M2.5 ($50/month)
- Moved heartbeats to local models (LM Studio)
- Result: Faster, cleaner, zero bankruptcies

## The "Freshman Rule"

| Rule | Description |
|------|-------------|
| **One task per agent** | If you give AI 5 jobs, it fails at 4 |
| **Treat like freshmen** | Don't assume they remember context. Give clear, ground-up instructions every time |

## The Hybrid Engine

### Workload Distribution
| Workload | Model | Why |
|----------|-------|-----|
| Heavy thinking | MiniMax (cloud) | Best reasoning |
| Heartbeats | Local (LM Studio) | Free, don't burn limits |
| Coding | Claude Code / Codex | Best for code |

### The Emergency Valve
> "When you add more agents, the 'Heartbeats' (constant check-ins) will start eating your rate limits. This is when you offload those boring, repetitive pings to LM Studio running locally. Use the local models as a secret valve to keep your MiniMax prompts reserved for actual coding and logic."

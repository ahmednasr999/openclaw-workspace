## Handover - silent reply filler root cause fixed
- Ahmed identified the real source of the `Nothing to add right now.` filler: the Telegram extension synthesizes a fallback `NO_REPLY` when a turn ends with no visible delivery, then runs it through outbound planning with direct silent-reply rewrite enabled, producing filler from `SILENT_REPLY_REWRITE_TEXTS`.
- Important correction to prior understanding: earlier preserve-silent-token patches in routed reply handling were not enough for this path because the synthetic Telegram fallback payload was fabricated inside the Telegram extension and did not carry the preserve marker.
- Source-level direct rewrite default was changed from `true` to `false` in all three required files: `/root/openclaw/src/shared/silent-reply-policy.ts`, `/root/openclaw/dist/silent-reply-CEfGkTRq.js`, and `/root/.openclaw/extensions/lossless-claw/node_modules/openclaw/dist/silent-reply-CEfGkTRq.js`.
- Production verification reported by Ahmed: after the patch, an explicit `sessions_yield` wait during a Kimi probe produced silence while waiting and no filler message appeared; the eventual real answer delivered cleanly.
- Operational tradeoff accepted: genuine silent-failure cases in direct messages now stay silent instead of emitting misleading filler. Ahmed prefers silence over fake text in this environment.
- Config cleanup: the now-redundant `agents.defaults.silentReplyRewrite.direct = false` override was removed from `openclaw.json` so the patched constant is the single source of truth.
- Rollback/reapply safety: pre-patch backups live at `/root/.openclaw/backups/silent-reply-direct-disable-2026-04-22/`, and runbook section 10.4 documents exact reapply `sed` commands plus a verification snippet.
- Durable warning: this behaves like the earlier lossless-claw artifact hotfix - any source rebuild, OpenClaw reinstall, or lossless-claw reinstall can wipe one or more patched files, so verify/reapply after rebuilds.
- Better upstream fix later should be narrower: teach the Telegram empty-turn fallback to distinguish `sessions_yield` with pending child work from true silent failure, so filler can remain available for the latter without appearing during intentional waits.
- Current handoff state from Ahmed: gateway healthy, all agents responding, lossless-claw 0.9.2 native, and no filler observed in production.

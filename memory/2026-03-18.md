### 03:17 AM — Full YouTube transcript captured: Cron Skills Pattern

**Source:** https://www.youtube.com/watch?v=Bjbl7I-1RtA by Marcelo (Clear Mud / Muddy OS)

**Complete Rules (Skills-First Checklist):**
1. Max 20 lines in cron prompt (ideally 5-11)
2. First line ALWAYS: "Read and follow [skill path]"
3. Update the SKILL.md, NOT the cron (only touch cron if schedule changes)
4. Never paste skill steps into the cron - reference the file always
5. One skill per workflow; multiple crons can reference same skill
6. Cron = scheduling + context. Skill = workflow + logic. Never mix.
7. Always use isolated sessions for crons (won't run if main session idle)
8. If you change a process, change the skill first

**Cron prompt template:**
```
Read and follow: ~/skills/[skill-name]/SKILL.md
Context:
- Project: ~/projects/[project]/
- Service: systemctl --user restart [service-name]
- Notify: [telegram-id or email]
- Data: ~/data/[relevant-file].json
```

**SKILL.md template:**
- Prerequisites (what must exist)
- Steps (numbered, detailed, no limit on lines)
- Validation rules and checks
- Output format and templates
- Edge cases and error handling
- Quality gates

**Do NOT put in SKILL.md:** schedule/time, environment-specific paths, model selection
**Do NOT put in cron:** workflow steps, business logic, output formatting

**His experience:** 52 active skills in Muddy OS. Morning briefs were inconsistent until moved to skill. Blog post output was inconsistent until skill-ified.

**Tip:** Run crons "wrong" for 1-2 weeks, notice inconsistencies, then dedicate a Saturday to converting all crons to skill-first pattern.

**Action for us:** Convert our top crons to skill-first. Check if our crons use isolated sessions.

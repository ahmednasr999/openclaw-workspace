**Script:** `/root/.openclaw/workspace/scripts/context-guardian.sh`
**Cron:** "Context Guardian - External Watchdog" (id: `65afe9ed-d5ec-4880-a034-f7d5efda3bc6`)
**Frequency:** Every 4 minutes
**Location:** Isolated agent (never runs in main session)

**How it works:**
1. Reads `openclaw status` to get current context %
2. If >= 75%:
   - Writes auto-flush marker to memory/YYYY-MM-DD.md
   - Commits all memory files: `git add -A && git commit -m "auto-flush: context guardian triggered at {PCT}%"`
   - Sends Telegram alert to Ahmed (message: "🚨 Context Guardian — Auto-Flush")
   - Sets 10-min cooldown lock to prevent alert spam
3. If < 75%: silently exits

**Log:** `memory/context-guardian.log` (one line per check, timestamp + context %)

**Why external?** At 91% context, NASR cannot read or process instructions. The enforcer must run outside the main session, check context from the OS, and act directly on files without needing NASR's participation.

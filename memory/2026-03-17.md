### Google Doc Premium Formatting: Full Rewrite

#### Split-Run Root Cause Found & Eliminated
- **Root cause:** Google Docs API sometimes splits text insertions into multiple "runs" at arbitrary boundaries (especially near smart quotes, Unicode chars, or after batch index drift). When styling was applied using calculated positions (`current_pos += len(text) + 1`), the positions drifted 1-3 characters from Google's actual indices, leaving fragments unstyled.
- **Example:** "Today's LinkedIn Pos" (bold) + "t" (no style) — API split at the smart quote boundary.

#### Full Document Cleanup
- Fixed 28 split-run issues across entire document (headings, alerts, body text, dividers)
- Fixed 3 divider lines wrongly styled as H1
- Removed duplicate Mar 15 section (6,120 chars deleted)
- Cleaned 17 markdown artifacts + 5 stray asterisks
- Applied uniform styling to all 6 days (Mar 12-17): blue date H1s, bold H2s, dark H3s, 10pt body, 6pt grey dividers
- Final audit: 0 markdown artifacts, 0 unlinked URLs, 0 split-runs, 0 wrong heading levels

#### Script Architecture Rewrite (PERMANENT FIX)
- **Old (vulnerable):** Insert text → calculate positions manually → style using calculated indices → drift → split-runs
- **New (immune):** 6-phase architecture:
  1. Insert raw text only (one batch call)
  2. Re-read document from Google to get ACTUAL paragraph indices
  3. Match paragraphs by text content (not position)
  4. Apply formatting using Google's own startIndex/endIndex
  5. Make URLs clickable using Google's own element indices
  6. Verification pass: re-read again, check every paragraph for size/color mismatches, auto-fix
- **Key principle:** Zero manual position math. Every index comes from Google's API response.
- Tested: regenerated Mar 17 section, Phase 6 verification passed with zero issues.
